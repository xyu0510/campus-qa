# 高德地图API密钥配置说明

## ❌ 问题：INVALID_USER_KEY

当前API密钥权限不足，无法使用路线规划功能。

## 🔧 解决方案

### 方案一：配置API密钥权限（推荐）

#### 1. 登录高德开放平台
```
https://console.amap.com/
```

#### 2. 进入应用管理
- 点击右上角"控制台"
- 选择"应用管理" → "我的应用"

#### 3. 添加新Key
- 点击"添加Key"
- 服务平台选择：**Web服务**
- 勾选以下服务：
  - ✅ Web服务API
  - ✅ JavaScript API
  - ✅ 路径规划
  - ✅ 地理/逆地理编码

#### 4. 获取新Key
- 提交后获取新的Key
- 复制Key值

#### 5. 更新代码
在 `map-navigation.html` 中替换API密钥：
```javascript
// 找到这一行
<script src="https://webapi.amap.com/maps?v=2.0&key=885e7d0e06d7bd72f0de74e1d6c5189b">

// 替换为你的新Key
<script src="https://webapi.amap.com/maps?v=2.0&key=你的新Key">
```

### 1. 登录高德开放平台
```
https://console.amap.com/dev/key/app
```

### 2. 查看Key详情
找到Key：`885e7d0e06d7bd72f0de74e1d6c5189b`

### 3. 检查服务平台
- 如果是"Web端(JS API)"，只能显示地图

### 4. 检查服务配置
确保勾选了：
- ✅ 地理/逆地理编码


## ⚠️ 注意事项

1. **API密钥安全**
   - 不要将密钥提交到公开仓库
   - 设置域名白名单限制使用范围
   - 定期更换密钥

2. **配额限制**
   - 个人开发者：每天10万次调用
   - 企业开发者：每天100万次调用
   - 超出配额会返回错误

3. **HTTPS限制**
   - Web服务API必须使用HTTPS
   - 本地测试可以使用HTTP
